# Phase 4 計画書 — 映像品質追求・自動ディレクションシステム

> 作成日: 2026-03-10
> 前提: Phase 1（コアエンジン9機能）+ Phase 2（品質評価+編集支援9機能）+ Phase 3（パイプライン統合）+ E2Eテスト完了済み

---

## 1. Phase 4 の位置づけ

Phase 1-3で「TEKO対談ディレクションレポート自動生成」の本番運用が可能になった。
Phase 4では、システムの本来の目的である **「映像品質を追求し続ける自己成長型システム」** を実現する。

具体的には以下の3軸:
1. **C-1/C-3の実映像対応** — スタブ実装から実測値への切り替え（opencv/ffmpeg）
2. **映像トラッキング+学習** — 外部映像の収集・分析・学習機能（NEW-4/5/6/7）
3. **管理+インフラ** — 編集者管理・巡回監査・通知・PDCA自動化（NEW-8, F-3, J-3/4/5/6）

## 2. Phase 4A: C-1/C-3 実映像対応（優先度: 高）

### 2-1. C-1 フレーム画像マルチモデル評価（opencv実装）

**現状**: 文字起こしベースの推定スコア（スタブ）
**目標**: 実際のフレーム画像を抽出→マルチモデルで評価

**実装計画**:
| ステップ | 内容 | 見積 |
|---------|------|------|
| 1 | opencv-python インストール + 動画ファイルパス解決 | 小 |
| 2 | 代表フレーム抽出（等間隔 + ハイライト時点） | 中 |
| 3 | Claude Vision API連携（フレーム画像 → 構図/照明/色バランス評価） | 中 |
| 4 | GPT-4o Vision API連携（同一フレーム → 独立評価） | 中 |
| 5 | マルチモデル合意判定ロジック（合意→指摘、不合意→要検討） | 小 |
| 6 | 推定値→実測値のグレースフル切り替え | 小 |

**依存関係**:
- AI開発5の動画ファイルパスへのアクセス（Googleドライブ or ローカルコピー）
- Claude API / OpenAI APIのVision対応エンドポイント
- opencv-python のインストール（`requirements.txt` で有効化）

**リスク**:
- 動画ファイルが大きい場合のフレーム抽出時間
- API費用（フレーム数 × モデル2種 × 動画数）

### 2-2. C-3 音声品質自動評価（ffmpeg実装）

**現状**: 文字起こしベースの推定スコア（スタブ）
**目標**: 実際の音声トラックを分析→品質評価

**実装計画**:
| ステップ | 内容 | 見積 |
|---------|------|------|
| 1 | ffmpeg/ffprobe インストール + 音声トラック抽出 | 小 |
| 2 | 音量レベル分析（LUFS測定、ピーク検出） | 中 |
| 3 | BGM/会話分離推定（スペクトル分析） | 大 |
| 4 | ノイズフロア検出 | 中 |
| 5 | 区間ごとの音量一貫性チェック | 中 |
| 6 | 推定値→実測値のグレースフル切り替え | 小 |

**依存関係**:
- ffmpeg のインストール（`brew install ffmpeg`）
- 動画ファイルへのアクセス

## 3. Phase 4B: 映像トラッキング + 学習（4機能）

### 3-1. NEW-4 映像トラッキング

**概要**: YouTube等から参考映像を収集・追跡
**実装方針**:
- YouTube Data API v3でチャンネル/プレイリスト登録 → 新着検出
- 対象: TEKO参考チャンネル、競合コンテンツ、映像品質の高いチャンネル
- メタデータ（タイトル、再生数、サムネイル）をJSON保存
- 格納先: `~/TEKO/knowledge/raw-data/video-direction/tracking/`

### 3-2. NEW-5 トラッキング映像分析

**概要**: 収集した映像の要素分解
**実装方針**:
- YouTube文字起こし取得（yt-dlp --write-auto-sub）
- 映像構成分析（イントロ長、カット頻度、テンポ変化）
- サムネイル分析（Vision API）
- 分析結果をナレッジDBに蓄積

### 3-3. NEW-6 人間FB学習

**概要**: なおと・パグさんのFBを構造化して蓄積
**実装方針**:
- FBフォーム（Telegram Bot or スプシ列）からの入力受付
- FB内容の構造化: {対象動画, 評価軸, 評価値, 自由記述}
- ディレクションルールへの自動反映（ルール優先度の調整）
- `docs/teko_interview_direction_manual.md` への学習結果追記

### 3-4. NEW-7 トラッキング映像学習

**概要**: 外部映像のパターンを学習 → ディレクション指示に反映
**実装方針**:
- NEW-5の分析結果からパターン抽出（テンポ、カット割り比率等）
- パターンDB化 + タグ付け（「Netflix風」「YouTube人気形式」等）
- direction_generator.pyへの追加ルール自動注入

## 4. Phase 4C: 管理 + インフラ（6機能）

### 4-1. NEW-8 編集者管理シート

**概要**: 編集者の名簿・スキル・工程・実績を一元管理
**実装方針**:
- Google Sheets「編集者管理」タブの新設（or 既存スプシに追加）
- SheetsManagerの拡張（読み書き）
- B-3スキルマトリクスとの自動連動

### 4-2. F-3 編集者引き継ぎパッケージ

**概要**: 担当者交代時の自動パッケージ化
**実装方針**:
- 対象編集者のFB履歴・修正回数・品質傾向をJSON集約
- HTML引き継ぎレポート自動生成
- direction-pagesに公開

### 4-3. J-3 定期巡回監査

**概要**: パイプライン死活監視 + 品質異常値検出
**実装方針**:
- launchd 15分間隔で巡回
- チェック項目: AI開発5新規ファイル検出、未処理動画滞留、品質スコア異常値
- 異常検出時にTelegram通知（J-4連携）

### 4-4. J-4 Telegram/LINE通知

**概要**: レポート完成時 + 品質警告の自動配信
**実装方針**:
- AI開発6のTelegram Bot基盤を流用
- 通知トリガー: レポート生成完了、品質スコアC以下、未処理滞留24時間超
- LINE連携はAI開発9のLINEエージェント経由

### 4-5. J-5 品質改善ループ

**概要**: ディレクション→編集→評価→FB→ルール更新のPDCA自動化
**実装方針**:
- ステートマシン: DIRECTED → EDITING → REVIEW → FEEDBACK → RULE_UPDATE
- 各状態遷移のトリガーとアクション定義
- B-2ダッシュボードへの改善率自動追記

### 4-6. J-6 マルチMac分散処理

**概要**: 重い処理をMac2/3に分散
**実装方針**:
- mission-dispatch.sh連携（タスク指示書配置方式）
- フレーム抽出・音声分析等の重いタスクをMac2/3にオフロード
- 結果のrsync回収 + 統合
