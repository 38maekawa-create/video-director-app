# タスク指示書: ディレクションレポート全件監査+修正

## 目的
GitHub Pages（https://38maekawa-create.github.io/direction-pages/）で公開中のディレクションレポート32件に名称誤り・空ページ・表記ブレが多数あり、なおとさんから全件監査+修正の指示が出た。

## 背景
AI開発10の映像品質エージェントがE2Eで生成したディレクションレポートを実際になおとさんが確認したところ、以下の不備が指摘された:
1. メンバーの名称が間違いだらけ・誤字が多い
2. 個別ページに進むと内容が空のものがある可能性
これは実運用に直結する品質問題であり、最優先で修正が必要。

## 全体工程における位置づけ
レポート品質の修正。修正後になおとさんが再確認し、パグさん・編集者に共有する流れ。

## 監査手順

### Step 1: 全32件のindex.htmlから名前を全抽出
- GitHub Pagesのindex.htmlをパースし、全ゲスト名を一覧化

### Step 2: MEMBER_MASTER.jsonとの突合
- `~/TEKO/knowledge/people/MEMBER_MASTER.json` を読み込む
- 各レポートのゲスト名がMEMBER_MASTER.jsonの正式名と一致しているか全件チェック
- 不一致リストを作成

### Step 3: 個別ページの内容監査
- 全32件の個別HTMLページにアクセスし、以下を確認:
  - ページが空でないか（本文が存在するか）
  - ゲスト名が正しいか
  - プロフィール情報が正確か（年齢、職業等）
  - 演出ディレクションが存在するか

### Step 4: 修正実施
- `~/AI開発10/output/` 配下のHTMLファイルを修正
- ゲスト名の修正（ファイル名 + HTML内の全箇所）
- 空ページがあれば再生成
- index.htmlの名前も修正

### Step 5: direction-pagesリポジトリへpush
- 修正後のHTMLを `direction-pages` リポジトリにcommit + push
- GitHub Pagesで反映を確認

### Step 6: 検証レポート出力
- `output/DIRECTION_PAGES_AUDIT_REPORT.md` に以下を記載:
  - 全32件の監査結果（OK/NG + 修正内容）
  - 修正件数のサマリー
  - 残存する問題（あれば）

## 完了条件
1. 全32件のゲスト名がMEMBER_MASTER.jsonの正式名と一致していること
2. 「不明」「ゲスト氏」等のフォールバック名が0件であること
3. 空ページが0件であること
4. 修正後のHTMLがGitHub Pagesに反映されていること
5. 監査レポートが出力されていること

## 禁止事項
- 推測でゲスト名を決めない。MEMBER_MASTER.jsonに載っていない場合は「要確認」として残す
- レポート本文の内容（演出ディレクション等）を勝手に変更しない。名前の修正のみ
