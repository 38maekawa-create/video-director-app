## 3. 重大問題（対応必須）

### 3-1. ゲスト層分類の誤り（3件）

#### hiraiさん: 層b → 正しくは層a

- **現状**: プロファイル不足により「層b（デフォルト分類）」。confidence=low
- **原因**: ナレッジファイルのパースでプロファイル情報が取得できていない。`PersonProfile`のフィールドがすべて空
- **正解**: 年収850万（800万以上）+ 国内大手生保 → 層a。QUALITY_JUDGMENT_GUIDE.mdセクション1の具体例で「hirai: 850万 × 国内大手生保マーケ → 層a」と明記
- **影響**: 層bテンプレート（年収以外の強さの言語化）が適用され、本来のハイキャリア前面訴求の演出が行われない

#### さるビールさん: 層b → 正しくは層a

- **現状**: 「年収700万円（相対的強さの言語化が必要）」で層b判定
- **原因**: `_find_tier_a_keyword()`がプロファイルテキストから「キリン」を検出できていない。プロファイルの職種が「大手飲料メーカー勤務（営業企画）」で企業名が含まれていない
- **正解**: 年収700万 + キリンビール（日系超大手）→ 層a。QUALITY_JUDGMENT_GUIDE.mdで明記
- **影響**: 企業ブランドの権威性を活かした演出が欠落

#### 真生さん: 層b → 正しくは層c

- **現状**: プロファイル不足により「層b（デフォルト分類）」。confidence=low
- **原因**: hiraiさんと同様、ナレッジファイルのパースでプロファイル情報が取得できていない
- **正解**: 家族経営の会社の2代目 → 層c（自営業家系）。タイトルには「薬剤師×家族経営2代目」と正しく記載されているのに層分類が反映されていない
- **影響**: 層cの「自営業家系の背景やストーリーに重きを置く」演出テンプレートが適用されない

### 3-2. メンイチさんのハイライト品質問題

- **現状**: 冒頭ハイライト[01:44]に「大学の友達の親がやっぱり不動産やっててすっげえお金持ってて。でその友達がもうすごいお金使ってて、まあ自分で会社買ってもらったりとか。ブランドのバッグ持って、プレーた自由なんだよねみたいな」が選定されている
- **問題**: QUALITY_JUDGMENT_GUIDE.md セクション5のNGパターンに該当:
  1. 第三者エピソードで本人のフックとして整合性が取れない
  2. 1フレーズで意味が通じない（脈絡なく出てきて何の話か不明）
  3. パンチ力が弱い
- **なおとさんの指摘（03-20）**: このまさにこのハイライトについて品質問題を指摘済み
- **代替候補**: [18:55]「今月はモロモロ込みで200万ぐらいした」（実績数字 = 副業200万 → 不動産対談のフックとして最適）

### 3-3. ひろきょうさんのタイトル品質問題

- **現状**: タイトル案3つ全てが同一パンチライン「年収今800万ぐらい。ホワイトです。休みも取れますし、全体の仕事量もそんなにきつくもないです。」をそのまま使用
- **問題**:
  1. パンチラインが長すぎる（フック力がない）。1フレーズで刺さる凝縮された言葉ではない
  2. 3案がバリエーションではなくほぼ同一（「属性固定・パンチライン変更」の生成方法に違反）
  3. 発言そのままの引用で、フックとして加工されていない
- **QUALITY_JUDGMENT_GUIDE.md基準**: 「属性は固定して、パンチライン部分のバリエーションを3-5案出す」が正しい生成方法

## 4. 改善推奨（対応推奨）

### 4-1. PAYさん・くますけさんの層a判定理由の不十分さ

- **PAYさん**: 「年収850万円（800万以上で層a相当）」で判定。正しいが、LINEヤフー（日系超大手）の企業ブランド権威性が検出されていない
  - 原因: プロファイルの職種が「IT業界（街を便利にする仕組みを提供する誰もが知る大手企業）」で「LINE」「ヤフー」の文字列が含まれていない
  - 影響: 800万以上の閾値で拾えているため結果は正しいが、企業名が演出に反映されない
- **くますけさん**: 「年収870万円（800万以上で層a相当）」で判定。正しいが、元リクルート（日系超大手）の権威性が検出されていない
  - 原因: プロファイルの職種が「人材業界 企画職」で「リクルート」が含まれていない。タイトルには「前職リクルート」と正しく記載されている
  - 影響: タイトル生成では活用されているが、演出テンプレートの判定には反映されていない

### 4-2. さといも・トーマスさんの層分類

- **現状**: 層b判定（プロファイル年収が「高年収（具体額は非開示）」で数値抽出不可）
- **実態**: BIG4系列グループ会社勤務 + 年収1000万実態（なおとさん確認済み）→ 層a
- **ただし**: 不動産対談（teko_realestate）のため層分類の厳密性は低い（QUALITY_JUDGMENT_GUIDE.md記載）。タイトル・ハイライトは不動産実績にフォーカスしており適切
- **推奨**: プロファイルの年収フィールドに実態年収を反映するか、不動産対談は層分類をスキップするロジックの検討

### 4-3. プロファイル未取得問題（2件: hiraiさん、真生さん）

- ナレッジファイルから`PersonProfile`が正常にパースできていない2件は、層分類だけでなくハイライト選定・演出テンプレートの品質にも影響する
- ナレッジファイルのMarkdown構造がパーサーの期待するフォーマットと合っていない可能性。要調査

### 4-4. 概要欄の未生成（11件/29件 = 38%）

- 概要欄が空または未生成のプロジェクトが11件ある
- テンプレート固定部分＋AI生成部分（ハッシュタグ・タイムスタンプ）の完全な概要欄が生成されているのは約18件
- 概要欄の生成はdescription_writer.pyで行われるが、バッチ再生成で全件に適用されていない可能性

### 4-5. しおさん・けーさんの全体品質

- **しおさん（QS:75）**: タイトル案1「ベストフィットの環境が痒くなる」は意味が曖昧で1フレーズで刺さらない。ハイライトのパンチライン選定も「ベストフィットの環境ならどこにでも根を張れる」等、抽象的な表現が多い
- **けーさん（QS:77）**: タイトル案にパターンB（パンチライン先頭型）のみで、パターンA（年収先頭型）がない。年収が700万未満のため年収先頭は不適切とも判断できるが、バリエーションとしてあってよい

### 4-6. ろくさんの年収パース問題

- プロファイルの年収: 「650-700万円、転職すれば800-1000万円程度を想定」
- 分類結果: 「層a/cの条件に該当せず」（confidence=low）
- 問題: _extract_income()が「650-700万円」の範囲表記から650のみ抽出しているか、または範囲表記を処理できず年収0として扱っている
- 28歳で650万なら層b。700万なら20代なので層a（20代+700万以上）の可能性あり。ただし「20代後半」の表記がプロファイルに含まれているかの確認が必要

## 6. 全体的な品質傾向

### 改善された点（Phase 3改修の成果）

1. **ゲスト層分類ロジック**: 5段階判定（年収1000万超/700万+企業ブランド/700万+20代/800万超/士業経営層）が正しく実装されている。全29件中24件が正しく分類された（正解率83%）
2. **タイトルの属性+パンチライン構造**: パターンA（年収先頭型）が正しく適用されている案が大半。属性の見せ方とパンチラインの分離が適切
3. **コンテンツライン判定**: 不動産対談2件（さといも・トーマス、メンイチ）が正しくteko_realestateに分類
4. **ハイライト選定**: 大半のプロジェクトで属性紹介・実績数字・パンチラインがバランスよく選定されている

### 残存する課題

1. **プロファイルパース失敗**: ナレッジファイルのフォーマット差異により2件（hiraiさん、真生さん）でプロファイルが取得できず、層分類がデフォルト値に

3. **ハイライト品質の自動QC不足**: メンイチさんのNGハイライトが自動検出されずに残っている。パンチライン品質の自動判定ロジックが必要
