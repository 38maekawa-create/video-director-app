# タスク指示書: 映像品質追求システムのアーキテクチャ設計

## 1. 目的
映像品質を追求し続ける自己成長型システムの全体アーキテクチャを設計する。TEKO対談動画のディレクションレポート自動生成を最初のユースケースとし、将来的に映像作品全般の品質追求に拡張する。

## 2. 背景
- AI開発5（動画ナレッジ）がコンテンツ・ナレッジ化軸でYouTubeトラッキング済み
- 本システムは「映像作品としての演出・デザイン・品質」軸でのトラッキング
- 動画編集者用マニュアル（~/TEKO/knowledge/raw-data/chatgpt/chatgpt_リール編集者_指示マニュアル_011.md）が既に存在
- カメラマンがGoogleドライブにアップ → AI開発5が自動処理 → 本システムがディレクションレポート生成

## 3. 設計検討事項

### データフロー
- AI開発5からの入力（文字起こし、整形済みナレッジ、メタデータ）
- 映像分析の方法（ffmpegでのフレーム抽出 → Claude Vision API、またはスクリーンショットベース）
- ディレクションレポートの出力形式（Markdown? PDF? Googleドキュメント?）

### 映像品質トラッキング
- どのプラットフォームの映像をトラッキングするか（YouTube, Instagram Reels, TikTok）
- 品質評価の軸（カット割り、色彩、テロップ、BGM、テンポ、画角変化等）
- トラッキングした映像の分析をどう自動化するか

### 自己成長メカニズム
- 良い映像パターンのナレッジベース構築
- ディレクションレポート → 編集結果 → フィードバックループの設計
- 時間経過によるトレンド変化への追従

### AI開発5との連携インターフェース
- AI開発5のどの出力を入力として受け取るか
- トリガー方法（AI開発5の処理完了をどう検知するか）
