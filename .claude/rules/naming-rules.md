# AI開発10 命名・ファイル管理ルール

> 兵隊（Claude Code CLI）が自動参照する。ファイル操作・命名時に必ず従うこと。

## Drive素材のファイル命名ルール

- **C番号昇順で連番**: C001, C002, C003...（撮影順がわかるようにする）
- **分割ファイルの連番**: C001_1, C001_2, C001_3...（同一撮影の分割）
- **🔴「短尺」禁止**: ファイル名に「短尺」を使わない。C番号の連番で管理する
- **理由**: C番号昇順で撮影順が自然にわかる。「短尺」では順序が不明

## 出力先ルール

| 種別 | 出力先 |
|------|--------|
| ディレクションレポート | `output/reports/` |
| トラッキング結果 | `output/tracking/` |
| ログ・エラーログ | `.data/` |
| 生データ | `~/TEKO/knowledge/raw-data/video-direction/` |

## ゲスト名の表記

- ゲスト名はナレッジ（MEMBER_MASTER.json — member_master.py経由、Supabase）と一致させる
- 表記揺れ発見時は KNOWN_CORRECTIONS.md に記録してから修正
- 名前誤表記は致命的バグ扱い（jp#18: 生成物に人名が含まれる場合は必ずナレッジで確認）
