# タスク指示書: 映像品質エージェント アプリ化（Phase 5）

## 目的
映像品質エージェントのユーザー体験を形にする。
ディレクター（なおとさん）がスマホから音声でフィードバックし、それをエージェントが「編集者が気持ちよく動ける指示」に変換してVimeoレビューに投入する体験を実現する。

## 背景
AI開発10はPhase 4Bまで完了し、コア機能（分析・レポート生成・トラッキング・学習）は動いている。
しかしユーザー体験としてはGitHub Pages + スプシのみで、フィードバック入力の体験が未整備。
なおとさんから「PCで動画を流しながら、スマホで音声フィードバック → エージェントが変換 → 編集者に届く」という構想が出た。

## 全体工程における位置づけ
Phase 1-4B（エンジン完成）→ **Phase 5（アプリ化・UX）** → Phase 6（Vimeo API実連携・本番運用）
今回はPhase 5。UIの見た目とナビゲーション + 音声入力フローの設計。データはモック（ハードコード）でOK。

## エージェントの3つの仕事

### 1. 翻訳（言い回しの最適化）
- ディレクターの本音（「マジでセンスない」）を、編集者が気持ちよく受け取れる表現に変換する
- 角が立たず、かつ的確な指示に書き換える
- 元の意図は絶対に変えない。トーンだけ変える

### 2. 具体化（ナレッジ引用）
- 蓄積された良い映像のパターンDBから、「こうすべき」の参考事例を自動引用する
- 「このテロップの色は、[参考動画名]の12:15の表現を参照して、コントラスト比の高い白縁+ネイビーにすると良い」のように具体的な改善案を出す
- Phase 4Bの映像トラッキング+学習で蓄積されたデータを活用する

### 3. 構造化（タイムスタンプ・要素・優先度の自動整理）
- 音声入力からタイムスタンプを抽出する
- 対象要素（テロップ/カット/BGM/色味/テンポ等）を自動分類する
- 優先度（高/中/低）を自動判定する
- Vimeoレビューモードに投入可能なフォーマットに整形する

## 画面仕様

### 1. プロジェクト一覧画面
- 動画プロジェクトをカード表示（サムネイル、タイトル、ステータス）
- ステータス: ディレクション済 → 編集中 → レビュー待ち → 公開
- 未レビューの動画にバッジ表示

### 2. ディレクションレポート画面
- AI開発10が生成したディレクションレポートの表示
- ゲスト分類、年収演出判断、カットポイント提案の一覧
- 「この方針でOK」「修正指示」ボタン

### 3. 音声フィードバック画面（メイン機能）
- 動画のタイムライン表示（Vimeoプレーヤー埋め込み or サムネイルタイムライン）
- 録音ボタン（長押し or トグル）
- 音声 → 文字起こし → 構造化のリアルタイムプレビュー
- 変換後のフィードバック（エージェントが書き換えた版）のプレビュー
- 「この内容で送信」「編集する」ボタン
- 送信先: Vimeoレビュー / 編集者チャット

### 4. フィードバック履歴画面
- 動画ごとのFB一覧（タイムスタンプ順）
- 元の音声 / 変換後テキスト / 編集者の対応状況
- 学習効果の可視化（「前回のFBが次の動画に反映された」等）

### 5. 品質ダッシュボード画面
- 動画ごとの品質スコア推移グラフ
- 編集者別スキルマトリクス
- 品質アラート一覧

## デザイン方針
- ダークモード基調
- カラーアクセント: プロフェッショナルブルー（映像制作ツールの雰囲気）
- フォント: SF Pro（iOS標準）
- 録音UI: 大きく目立つ、片手操作可能
- iPhone SE〜iPhone 16 Pro Max対応

## 技術仕様
- Xcode プロジェクト名: `VideoDirectorAgent`
- iOS Deployment Target: 17.0
- SwiftUI + Swift 6
- MVVM構造
- モックデータは `MockData.swift` に集約
- 音声入力はモック（実際のWhisper連携はPhase 6）

## 完了条件と検証
1. Xcodeプロジェクトが正常にビルドできる（Simulator or Preview）
2. 上記5画面が実装されている（モックデータ）
3. 音声フィードバック画面で「録音 → 文字起こしプレビュー → 変換プレビュー → 送信」のフローが体験できる（モック）
4. 画面遷移が自然に動作する
5. PROGRESS.mdを更新する

## 禁止事項
- ネットワーク通信の実装（Phase 6以降）
- 実際のAPI呼び出し（Whisper, Vimeo, Claude等）
- 過度な装飾やアニメーション
- UIKit使用（SwiftUI純粋で）
