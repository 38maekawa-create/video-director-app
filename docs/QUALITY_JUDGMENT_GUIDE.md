## 1. ディレクションレポート — ゲスト層分類の判断基準

### ルール

| 条件 | 層分類 | 年収演出 |
|------|--------|---------|
| 年収1000万以上 | 層a | ON |
| 年収700万以上 + 企業ブランドの権威性 | 層a | ON |
| 年収800万以上（企業ブランドなしでも） | 層a | ON |
| 年収700万以上 + 20代 | 層a | ON |
| 士業・経営層の肩書き（年収不問） | 層a | ON |
| 年収700万以上（上記以外） | 層b | ON |
| 年収700万未満 | 層b | OFF |
| 自営業家系の2代目・3代目 | 層c | 別基準 |

### 背景やり取り（なぜこのルールになったか）

**元々のコード**: 年収1000万以上でのみ層a判定。700万以上900万未満はすべて層bに分類されていた。

**なおとさんの指摘（03-19）**:
> 「元リクルートで30代前半で年収900万弱あるくますけさんや年収900万のkosさんが層cに分類されてて、年収1000万なくて家業の後継者じゃない人は一律で層cに振り分けられてるのも俺のマニュアル参考になってなくない？！」

**企業ブランドの「日系超大手」の粒度（なおとさんの判断）**:
> 「アンディなんて名前出せないけど花王だし」
> 「キリンのさるビールも日系超大手枠にはいってるかな？！」
> 「LINEyahooとかもいるよ！payさんはそれ！」

→ 基準は「誰もが名前を知ってる企業」。GAFA+コンサルBIG4だけでなく、花王・キリン・リクルート・LINEヤフー等の日系超大手も含む。

**年齢×年収は層分類ではなく演出の強弱に使う（なおとさん確認済み03-19）**:
> マニュアルではないべいさん（年収600万×新卒2年目）は層bに分類されている。年齢×年収は「テロップで年次補足して凄さを際立たせる」演出判断であって、層分類の判定ロジックではない。

**さといも・トーマスの判断（なおとさん03-19）**:
> 「さといも・トーマスも名前出せないけどBIG4系列のグループ会社で年収1000万いってるからね！」
→ プロファイル上は「年収非開示」だが実態は層a。

## 2. ディレクションレポート — 演出ディレクションの品質基準

### ルール

編集者が読んで**即座にアクションに移せる具体性**が必須。以下はNG:
- 「色変え: テロップ出現に合わせて画面演出（フラッシュ or 背景色変更）」→ 何色に変えるか不明
- 「属性テロップ: ゲストの肩書き・属性情報を表示」→ 何を表示するか不明
- 相槌（「そうですね」「ああ」）をパンチラインに選定

OKな例:
- 「数字テロップ: 「1100万」を画面の30%以上のサイズで表示。フォント: 太ゴシック、色: ゴールド。表示タイミングは発言と同時、3秒間キープ」
- 「画角: 発言開始の1秒前にワイド→寄り（バストアップ）にカット切替。話者の表情が画面の60%以上を占める構図」

### 背景やり取り

**なおとさんの指摘（03-19）**:
> 「ディレクションレポートが俺のマニュアル通りに作られてない気がして」

マニュアル（teko_interview_direction_manual.md）では:
- りょうすけさんの例: 「凸版印刷（業界最大手）に新卒入社」「次の転職で年収1300万狙えるポジション」と**具体的にどこをフォーカスするか**まで書いてある
- ゆりかさんの例: 「週4本業500万＋在宅副業月数十万という働き方を強さとして見せる」

レポートはこのレベルの具体性に達していなかった。テンプレ的な抽象指示の羅列になっていた。

## 3. YouTubeタイトル生成 — 設計思想と品質基準

### パターン定義

- **パターンA（年収先頭型・推奨/最近の主流）**: `年収[金額][年代][職業]「[パンチライン]」[名前]さんが語る[テーマ]とは【TEKO独占インタビュー】`
- **パターンB（パンチライン先頭型・初期の動画）**: `「[パンチライン]」[年代]年収[金額]円[職業]の[名前]さんが語る[テーマ]とは？【TEKO独占インタビュー】`

### 設計思想（なおとさんの原文・03-18セッション）

**属性を見せる2つの目的:**

> 「年収や社格などの属性の強さをシンプルにアピールすることで、ターゲットへ訴求したりフックを強めたいという狙いがあるのと、属性の強さを提示することで、我々がターゲットとしている層に「これは自分のためのコンテンツなんだ」という共感を抱かせることが、まず一つ目の目的です。」

> 「属性の中でも年齢であったり、性別であったり肩書き（ママ、バリキャリ女性、父親、家族経営の会社の2代目）といった要素を見せることで、同じ立ち位置や同じ属性の人に対して共感を抱かせること、より「自分ごと」として物事を捉えてもらう、あるいはこのインタビューコンテンツを「自分と近しい境遇の人だから」という理由で視聴するきっかけにしたい、という意図があります。」

**パンチラインの役割:**

> 「パンチラインに関しては、シンプルに訴求文としてフックにしたいと考えています。具体的には、視聴するインセンティブを生むフックとしての役割や、ターゲットにしっかりと「刺しに行く」という意図です。」

**判断軸:**

> 「属性（年収・社格・年齢などデモグラ情報）とパンチラインだけ見せられたらあとは見切れていいやって判断軸でタイトルのコピーライティング作ってる」

### パンチラインの品質基準

パンチラインは**本人の肉声を1フレーズ抽出**。感情・価値観・覚悟が伝わる言葉。

**属性は固定して、パンチライン部分のバリエーションを3-5案出す**のが正しい生成方法。

### なおとさんの承認例（03-18）

> なおとさん「めちゃいい！！！！そうこれ！これよ！！」
> なおとさん「全員震えるくらいいいね、これだよこれ」
> なおとさん「スリマンさんとか、「年収2000万でも足りないかな」とかにするともっといいし」

### パターンA/Bの誤りの経緯

> なおとさん「パターンA（パンチライン先頭型・推奨/最近の主流）、パターンB（年収先頭型・初期の動画）— これ、最近の主流と初期が反対だね！」

→ 修正: パターンA = 年収先頭型（推奨/最近の主流）、パターンB = パンチライン先頭型（初期）

## 4. 概要欄文章生成

概要欄はタイトルの設計思想と連動。属性情報 + テーマの概要 + CTA（LINE誘導等）の構成。
品質基準はタイトル生成と同じ設計思想に基づく。

※概要欄に関する詳細な壁打ちは03-15〜19のセッションでは発生していないため、タイトルの設計思想を準用する。

## 5. このガイドの使い方

1. **新しいゲストが来た時**: セクション1の分類例を参照し、類似ケースで判断する。判断に迷う場合は「誰もが名前を知ってる企業か」「年収700万以上か」で判定
2. **ディレクションレポート生成時**: セクション2の品質基準を確認。テンプレ的な抽象指示になっていないか自己チェック
3. **タイトル生成時**: セクション3の設計思想を読んでからプロンプトを実行。属性を見せる2つの目的を理解した上で生成する
4. **判断に迷ったら**: 背景やり取りを読んで、なおとさんの判断パターンから類推する
